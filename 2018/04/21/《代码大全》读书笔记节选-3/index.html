《代码大全》读书笔记节选(3)

因为前几学期挂科有点严重，所以最近要开始准备重修考试，看书的进度更慢了…

第6章 可以工作的类

6.1

  • 介绍 ADT，很简单，快速过一遍，或者去看萧井陌b站直播，有助于理解。

  • 从 ADT 正式引入 类 的概念，增加了许多代码实例，看着比较舒服。

  • 举例的同时还举了一个反例：如果在某些情况下没有使用ADT，那么就会出现幻数/命名冲突/缺少一致性/高耦合等问题。

  • 了解ADT的益处，一是可以更好地了解ADT的使用场景，二是能迫使我们习惯ADT。

  • 接下来又是一堆实例，在以后学习/工作中如果遇到相关问题，可以在这部分寻找解决问题的线索。

6.2

  • 从ADT的概念逐步将相关元素组合为一个完整的类，本节主要讲解该过程的第一个步骤：创建类的接口

  • 其实就是讲编程风格，过一下Google的编程风格文档应该就差不多了。因为我在近一年学习的过程中，几乎没怎么碰过C/C++/Java，所以手里也没有相关的代码可以边看边改，这部分也就略读过去了。

  • 封装：阻止你看到细节。抽象：让你忽略实现细节的模型来管理复杂度。所以说封装是比抽象更强的概念。简单来讲，封装就是 Python Class 里的 dunder method，抽象的话例子实在太多就不说了。

  • 书上这一块涉及到了挺多C/C++开发相关知识，当前暂时用不到，就跳过了。

6.3

  • 用 has a 和 is a 分别区分包含/继承。用简单的单词/语法来区分一些概念固然是不错的办法，但是这样做最大的缺点就是可拓展性太差，随便举个反例：《Python基础教程》的OOP部分，可以用生拉硬套来形容。

  • 警惕有超过7个数据成员的类。同时避免让继承体系过深，把继承层次限制在最多6层之内（其实2-3层就差不多了）。

  • Liskov 替换原则。

  • 让所有数据都是 private。

  • 多重继承主要用途是定义 mixins，也就是说使用背景必须清晰明了而且几乎不可替代，简单来讲就是很自然。

  • 后面讲的一些内容和之前的比较相似，可以说都在围绕KISS原则。

  • 应该避免的类部分内容提醒我们避免过度工程。
